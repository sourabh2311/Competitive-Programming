    This task is about finding an optimal schedule for nn jobs on a single machine, if the job ii can be processed in titi time, but for the tt seconds waiting before processing the job a penalty of fi(t)fi(t) has to be paid.

    Thus the task asks to find such an permutation of the jobs, so that the total penalty is minimal. If we denote by ππ the permutation of the jobs (π1π1 is the first processed item, π2π2 the second, etc.), then the total penalty is equal to:

    F(π)=fπ1(0)+fπ2(tπ1)+fπ3(tπ1+tπ2)+⋯+fπn(∑i=1n−1tπi)F(π)=fπ1(0)+fπ2(tπ1)+fπ3(tπ1+tπ2)+⋯+fπn(∑i=1n−1tπi)

    Solutions for special cases

    Linear penalty functions

    First we will solve the problem in the case that all penalty functions fi(t)fi(t) are linear, i.e. they have the form fi(t)=ci⋅tfi(t)=ci⋅t, where cici is a non-negative number. Note that these functions don't have a constant term. Otherwise we can sum up all constant term, and resolve the problem without them.

    c⋅tπi+1−cπi+1⋅tπi≥0∀i=1…n−1c⋅tπi+1−cπi+1⋅tπi≥0∀i=1…n−1

    Thus we obtain the optimal schedule by simply sorting the jobs by the fraction citiciti in non-ascending order.

    It should be noted, that we constructed this algorithm by the so-called permutation method: we tried to swap two adjacent elements, calculated how much the penalty changed, and then derived the algorithm for finding the optimal method.

    Exponential penalty function

    Let the penalty function look like this:

    fi(t)=ci⋅eα⋅t,fi(t)=ci⋅eα⋅t,
    where all numbers cici are non-negative and the constant αα is positive.

    By applying the permutation method, it is easy to determine that the jobs must be sorted in non-ascending order of the value:

    vi=1−eα⋅ticivi=1−eα⋅tici

    Identical monotone penalty function

    In this case we consider the case that all fi(t)fi(t) are equal, and this function is monotone increasing.

    It is obvious that in this case the optimal permutation is to arrange the jobs by non-ascending processing time titi.

    The Livshits-Kladov theorem

    The Livshits-Kladov theorem establishes, that the permutation method is only applicable for the above mentioned three cases, i.e.:

    • Linear case: fi(t)=ci(t)+difi(t)=ci(t)+di, where cici are non-negative constants,
    • Exponential case: fi(t)=ci⋅eα⋅t+difi(t)=ci⋅eα⋅t+di, where cici and αα are positive constants,
    • Identical case: fi(t)=ϕ(t)fi(t)=ϕ(t), where ϕϕ is a monotone increasing function.

    In all other cases the method cannot be applied.

    The theorem is proven under the assumption that the penalty functions are sufficiently smooth (the third derivatives exists).

    In all three case we apply the permutation method, through which the desired optimal schedule can be found by sorting, hence in O(nlogn)O(nlog⁡n) time.
