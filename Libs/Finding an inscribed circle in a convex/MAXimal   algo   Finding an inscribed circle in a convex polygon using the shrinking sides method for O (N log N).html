Алгоритм Algorithm

Итак, дан выпуклый многоугольник. Thus, a convex polygon is given. Начнём одновременно и с одинаковой скоростью сдвигать все его стороны параллельно самим себе внутрь многоугольника: Let's start simultaneously and with the same speed to move all its sides parallel to ourselves inside the polygon:

Пусть, для удобства, это движение происходит со скоростью 1 координатная единица в секунду (т.е. время в каком-то смысле равно расстоянию: спустя единицу времени каждая точка преодолеет расстояние, равное единице). Suppose, for convenience, this motion occurs at a rate of 1 coordinate unit per second (ie, the time is in some sense equal to the distance: after a unit of time, each point will overcome a distance equal to one).

В процессе этого движения стороны многоугольника будут постепенно исчезать (обращаться в точки). In the process of this movement, the sides of the polygon will gradually disappear (refer to points). Рано или поздно весь многоугольник сожмётся в точку или отрезок, и этот момент времени Sooner or later the entire polygon will shrink to a point or a segment, and this point in time будет являться ответом на задачу — искомым радиусом (а центр искомой окружности будет лежать на этом отрезке). will be the answer to the problem - the required radius (and the center of the required circle will lie on this segment). В самом деле, если мы сжали многоугольник на толщину In fact, if we squeezed the polygon by the thickness по всем направлениям, и он обратился в точку/отрезок, то это означает, что существует точка, отстоящая от всех сторон многоугольника на расстоянии in all directions, and it turned into a point / segment, it means that there is a point that is spaced from all sides of the polygon at a distance , а для бОльших расстояний — такой точки уже не существует. , and for larger distances, such a point does not exist any more.

Итак, нам надо научиться эффективно моделировать этот процесс сжатия. So, we need to learn how to effectively model this compression process. Для этого научимся для каждой стороны определять время , через которое она сожмётся в точку. For this we will learn for each side to determine the time through which it will shrink to a point.

Для этого рассмотрим внимательно процесс движения сторон. To do this, let us consider carefully the process of moving the sides. Заметим, что вершины многоугольника всегда движутся по биссектрисам углов (это следует из равенства соответствующих треугольников). Note that the vertices of the polygon always move along the bisectors of the angles (this follows from the equality of the corresponding triangles). Занесём эти времена для каждой стороны в некую структуру данных для извлечения минимума , например, красно-чёрное дерево ( We will enter these times for each side into a certain data structure to extract a minimum , for example, a red-black tree ( в языке C++). in the C ++ language).

Если многоугольник ещё не сжался в точку/отрезок, то эту сторону надо удалить из многоугольника, и продолжить алгоритм для оставшихся сторон. If the polygon has not yet contracted to a point / segment, then this side must be removed from the polygon, and continue the algorithm for the remaining sides. При удалении стороны мы должны соединить друг с другом её левого и правого соседа, продлив их до точки их пересечения. When deleting a party, we must connect to each other its left and right neighbors, extending them to the point of their intersection. При этом необходимо будет найти эту точку пересечения, пересчитать длины двух сторон и их времена исчезновения. It will be necessary to find this intersection point, recalculate the lengths of the two sides and their disappearance times.

При реализации для каждой стороны придётся хранить номер её правого и левого соседа (тем самым как бы построив двусвязный список из сторон многоугольника). At implementation for each party it is necessary to store number of its right and left neighbor (thereby as though having constructed a doubly connected list from sides of a polygon). Это позволяет реализовать удаление стороны и связывание двух её соседей за This allows you to implement the removal of the party and the binding of its two neighbors for . .

Если при удалении стороны оказывается, что её стороны-соседи параллельны , то это означает, что многоугольник после этого сжатия вырождается в точку/отрезок, поэтому мы можем сразу останавливать алгоритм и возвращать в качестве ответа время исчезнования текущей стороны (так что проблем с параллельными сторонами не возникает). If, when the side is removed, its side-neighbors are parallel , it means that the polygon after this compression degenerates to a point / segment, so we can immediately stop the algorithm and return as a response the disappearance time of the current side (so that problems with parallel sides does not arise).

Если же такая ситуация с параллельными сторонами не возникает, то алгоритм доработает до момента, в который в многоугольнике останется только две стороны — и тогда ответом на задачу будет являться время удаления предыдущей стороны. If this situation does not occur with parallel sides, then the algorithm will finish until the moment when only two sides remain in the polygon - and then the answer to the task will be the time of removal of the previous side.

Реализация Implementation

Приведём реализацию описанного выше алгоритма. Let us describe the implementation of the algorithm described above. Данная реализация возвращает только радиус искомой окружности; This implementation returns only the radius of the desired circle; впрочем, добавление вывода центра окружности не составит большого труда. however, the addition of the derivation of the center of the circle will not be very difficult.

Данный алгоритм элегантен тем, что из вычислительной геометрии требуется только нахождение угла между двумя сторонами, пересечение двух прямых и проверка двух прямых на параллельность. This algorithm is elegant in that from computational geometry only the angle between two sides is required, the intersection of two straight lines and the verification of two lines for parallelism.

Примечание. Note. Предполагается, что подаваемый на вход многоугольник — строго выпуклый , т.е. It is assumed that the input polygon is strictly convex , i.e. никакие три точки не лежат на одной прямой. no three points lie on one line.

struct pt {
	double x, y ;
	pt ( )  { }
	pt ( double x, double y ) : x ( x ) , y ( y )  { }
	pt operator - ( const pt & p ) const {
		return pt ( x - p. x , y - p. y ) ;
	}
} ;
 
double dist ( const pt & a, const pt & b ) {
	return sqrt ( ( a. x - b. x ) * ( a. x - b. x ) + ( a. y - b. y ) * ( a. y - b. y ) ) ;
}
 
double get_ang ( const pt & a, const pt & b ) {
	double ang = abs ( atan2 ( a. y , a. x ) - atan2 ( b. y , b. x ) ) ;
	return min ( ang, 2 * PI - ang ) ;
}
 
struct line {
	double a, b, c ;
	line ( const pt & p, const pt & q ) {
		a = p. y - q. y ;
		b = q. x - p. x ;
		c = - a * p. x - b * p. y ;
		double z = sqrt ( a * a + b * b ) ;
		a / = z, b / = z, c / = z ;
	}
} ;
 
double det ( double a, double b, double c, double d ) {
	return a * d - b * c ;
}
 
pt intersect ( const line & n, const line & m ) {
	double zn = det ( n. a , n. b , m. a , m. b ) ;
	return pt (
		- det ( n. c , n. b , m. c , m. b ) / zn,
		- det ( n. a , n. c , m. a , m. c ) / zn
	) ;
}
 
bool parallel ( const line & n, const line & m ) {
	return abs ( det ( n. a , n. b , m. a , m. b ) ) < EPS ;
}
 
double get_h ( const pt & p1, const pt & p2,
	const pt & l1, const pt & l2, const pt & r1, const pt & r2 )
{
	pt q1 = intersect ( line ( p1, p2 ) , line ( l1, l2 ) ) ;
	pt q2 = intersect ( line ( p1, p2 ) , line ( r1, r2 ) ) ;
	double l = dist ( q1, q2 ) ;
	double alpha = get_ang ( l2 - l1, p2 - p1 ) / 2 ;
	double beta = get_ang ( r2 - r1, p1 - p2 ) / 2 ;
	return l * sin ( alpha ) * sin ( beta ) / sin ( alpha + beta ) ;
}
 
struct cmp {
	bool operator ( ) ( const pair < double , int > & a, const pair < double , int > & b ) const {
		if ( abs ( a. first - b. first ) > EPS )
			return a. first < b. first ;
		return a. second < b. second ;
	}
} ;
